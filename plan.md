1.	Перечень автоматизируемых сценариев.
•	Покупка тура дебетовой картой: покупатель заходит на сайт, вводит данные карты, оплачивает. (позитивный сценарий)
•	Покупка кредита: покупатель заходит на сайт, оформляет тур в кредит. (позитивный сценарий)
•	Активные кнопки «Купить» и «Купить в кредит»: переключение между кнопками происходит без ошибок. (позитивный сценарий)
•	При введении актуальных данных карты не возникает ошибок. (позитивный сценарий)
•	При покупке тура по дебетовой карте происходит ошибка: клиент вводит актуальные данные карты, средств на карте достаточно, но происходит отказ. (негативный сценарий)
•	При оформлении кредита на тур происходит отказ. (негативный сценарий)
•	Все поля пустые, нет возможности отправить заявку. (негативный сценарий)
•	Невалидный номер карты, то есть менее 16 цифр. (негативный сценарий)
•	CVC менее, чем три цифры. (негативный сценарий)
•	Пустое поле CVC. (негативный сценарий)
•	ФИО на кириллице, форма выдаёт ошибку. (негативный сценарий)
•	Пустое поле «Фамилия». (негативный сценарий)
•	Пустое поле «Имя». (негативный сценарий)
•	Пустое поле номер карты. (негативный сценарий)

Разработчики внесли изменения в интерфейс (например, поменяли кнопки местами). В данном случае тест не пройдёт проверку, даже если приложение работает без проблем.
